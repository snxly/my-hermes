Hermes — Adapter / Gateway / AIAgent 关系

三种实例的创建时机、数量关系、引用依赖

commit 41aba91

Adapter — 实例特性

数量：每个平台一个，N:1 关系（多个 Adapter 对应一个 GatewayRunner）
生命周期：GatewayRunner 启动时创建，随进程一直存在
职责：维持平台连接（WebSocket/Webhook），接收消息 → 转 MessageEvent → 回调
关键方法：set_message_handler(fn) — 把 GatewayRunner._handle_message 注册进来

GatewayRunner — 实例特性

数量：全局唯一
生命周期：Gateway 进程从启动到关闭
持有的实例：SessionStore, _agent_cache (OrderedDict), _session_db, 多个 Adapter
关键：不直接创建 AIAgent 给 Adapter 用，而是通过缓存按需获取。用 OrderedDict 做 LRU 淘汰，硬上限 + 空闲 TTL 双重保护。

AIAgent — 实例特性

数量：≤ 活跃会话数，每个 session_key 最多一个缓存实例
生命周期：首次消息时创建（缓存未命中），后续复用，直到被 LRU 淘汰或 TTL 超时清出
缓存 key：session_key（"agent:main:wecom:dm:XuLuYing"）
缓存 value：(AIAgent, config_signature, msg_count, session_id)
缓存作用：保留 frozen system prompt + tool schemas → LLM provider 的 prefix cache 命中，节约 ~10x 成本
