Session 生命周期

Hermes Gateway — session.py + hermes_state.py

路由决策逻辑

  • session_key 存在且活跃 → 继续
  • session_key 存在但 end_reason ≠ NULL → 恢复或新建
  • session_key 不存在 → 查询可恢复或新建
  • resume_pending 过期 → 自动重置
  • force_new (/new) → 直接创建新 session

并发安全

  • _inflight_sessions: single-flight 去重
  • _inflight_lock: 保护 dict 操作
  • _lock: 保护 _entries 内存路由表
  • WAL 模式 + 随机 jitter 写重试
  • _save_lock: 序列化 sessions.json 写入

数据持久化

  • sessions 表: 元数据 + 统计 + 结束原因
  • messages 表: 完整对话记录
  • session_model_usage: 按模型/provider 拆分用量
  • gateway_routing: session_key → entry 映射
  • sessions.json: 内存路由表的磁盘镜像
